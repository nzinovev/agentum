# Security Policy

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security problems.

Email the maintainer at the address listed on the GitHub profile with:

- a description of the issue and its impact;
- a minimal repro or proof of concept;
- affected versions/commits, if known.

You should receive an acknowledgement within a few days. **No bounty or reward
is offered or implied.** Coordinated disclosure is appreciated; if you report
in good faith you will not be named publicly without your consent, and you may
request acknowledgment in release notes if you'd like the credit.

## Scope

This policy covers the Agentum source in this repository. It does not cover
third-party dependencies (report those upstream) or self-hosted deployments
modified beyond this repository's code.

## Supported versions

Agentum is pre-1.0. Only the latest `main` and the most recent tagged release,
if any, receive security fixes.
